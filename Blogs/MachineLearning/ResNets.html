Paper Read Through: "Deep Residual Learning for Image Recognition"

Link to the paper.

Context:

An accepted paper from Microsoft Research to CVPR 2016. The problem they are trying to solve is the difficulty of training very deep networks.


Motivation:

One of the main problems at the time when this paper came out was the fact that it was empirically shown that deeper networks consistently outperform their more shallow counterparts. The issue with this was the vanishing gradient problem


Methods:

The idea is a relatively simple one. They implement "blocks" of with a variable number of operations within each block. They call these blocks residual blocks where each one is focused on learning some residual mapping of the entire underlying mapping to be learned. If \(x\) is the input, then \(y = F(x,{W_i}) + x\), where the function \(F(x,{W_i})\) represents the residual mapping to be learned. Note that if the input and output channel dimensions are not the same we can instead do \(y = F(x,{W_i}) + {W_s}x\) where \(W_s\) is a linear projection to match the dimensions. Note that the inclusion of \(W_s\) also introduces another trainable parameter.
The authors note that there are three different approaches when the input and output dimensions don't match for the shortcuts:
  1. We can zero pad the shortcut so it still performs identity mapping.
  2. Projections shortcuts are used for increasing dimensions and identity shortcuts used elsewhere.
  3. Everything is a projection shortcut.


Results:

The results are pretty self-explanatory. If you want to read about the specifics, feel free to look at the paper yourself and read through them. Basically, the main takeaway is that the networks with residual blocks implemented outperformed those that don't. They have superior training times as well as superior accuracy.
